# Início rápido

Este guia acompanha você na configuração de um ambiente de desenvolvimento Primebrick completo em poucos minutos.

## Pré-requisitos

Antes de começar, certifique-se de ter instaladas as seguintes ferramentas:

| Ferramenta | Versão mínima | Notas |
|------------|---------------|-------|
| **Node.js** | 20+ | Verifique com `node --version` |
| **pnpm** | 9+ | Verifique com `pnpm --version` |
| **Docker** | 24+ | Necessário para os serviços de infraestrutura (banco de dados, NATS, Casdoor) |
| **Git** | 2.40+ | Para clonar os repositórios |

### Instalação dos pré-requisitos

Se você ainda não tem `pnpm`, pode instalá-lo com:

```bash
npm install -g pnpm
```

Verifique se o Docker está em execução:

```bash
docker info
```

## 1. Clonar os repositórios

Primebrick é composto por vários repositórios. Para um ambiente de desenvolvimento completo, clone o
monorepo principal ou os repositórios individuais:

```bash
# Clonar o backend
git clone https://github.com/michaelsogos/primebrick-be-v3.git

# Clonar o frontend
git clone https://github.com/michaelsogos/primebrick-fe-v3.git

# Clonar os microsserviços (opcional para começar)
git clone https://github.com/michaelsogos/primebrick-us-v3.git
```

## 2. Instalar as dependências

Entre em cada repositório e instale as dependências com `pnpm`:

```bash
cd primebrick-be-v3
pnpm install

cd ../primebrick-fe-v3
pnpm install
```

## 3. Iniciar os serviços de infraestrutura

O backend requer alguns serviços de infraestrutura (banco de dados, NATS, Casdoor). Estes estão
definidos em um arquivo `docker-compose.yml` e podem ser iniciados com Docker:

```bash
cd primebrick-be-v3
docker compose up -d
```

Isso inicia em segundo plano:

- **PostgreSQL** — banco de dados principal
- **NATS** — message broker para a comunicação entre microsserviços
- **Casdoor** — Identity Provider para autenticação OIDC

Verifique se todos os serviços estão ativos:

```bash
docker compose ps
```

## 4. Configurar as variáveis de ambiente

Copie o arquivo `.env.example` para `.env` e modifique os valores se necessário:

```bash
cp .env.example .env
```

As variáveis principais incluem:

```bash
# Banco de dados
DATABASE_URL=postgresql://primebrick:primebrick@localhost:5432/primebrick

# NATS
NATS_URL=nats://localhost:4222

# Casdoor
CASDOOR_ENDPOINT=http://localhost:8000
CASDOOR_CLIENT_ID=xxx
CASDOOR_CLIENT_SECRET=xxx
```

## 5. Executar as migrações do banco de dados

Aplique o esquema do banco de dados executando as migrações:

```bash
pnpm run db:migrate
```

## 6. Iniciar o backend

```bash
pnpm run dev
```

O backend estará disponível em `http://localhost:3001`.

Verifique se a API responde:

```bash
curl http://localhost:3001/api/v1/health
```

## 7. Iniciar o frontend

Em um novo terminal:

```bash
cd primebrick-fe-v3
pnpm run dev
```

O frontend estará disponível em `http://localhost:5173`.

## Verificação da instalação

Abra o navegador em `http://localhost:5173`. Você deverá ver a página de login do Primebrick.
Use as credenciais padrão do Casdoor para acessar o backoffice.

## Resolução de problemas

### Docker não inicia os serviços

Certifique-se de que o Docker Desktop está em execução e de que as portas `5432` (PostgreSQL), `4222`
(NATS) e `8000` (Casdoor) não estejam já ocupadas por outros processos.

### O backend não se conecta ao banco de dados

Verifique se o PostgreSQL está ativo e se a `DATABASE_URL` no arquivo `.env` está correta.

### Erro "pnpm not found"

Instale pnpm globalmente com `npm install -g pnpm` ou use `corepack enable pnpm`.
