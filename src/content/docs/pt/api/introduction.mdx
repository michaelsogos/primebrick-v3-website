# Introdução à API

A API REST do Primebrick é a interface principal para interagir com o sistema. Todas as
requisições passam pelo backend Express, que atua como gateway único para o frontend, os
microsserviços e os clientes externos.

## URL base

A URL base da API depende do ambiente:

| Ambiente | URL base |
|----------|----------|
| Desenvolvimento local | `http://localhost:3001/api/v1` |
| Staging | `https://api.staging.primebrick.dev/api/v1` |
| Produção | `https://api.primebrick.dev/api/v1` |

Todos os endpoints são servidos sob o prefixo **`/api/v1/`**. A versão da API faz parte da
URL, garantindo que mudanças não retrocompatíveis sejam introduzidas sob um novo prefixo
(ex. `/api/v2/`) sem interromper os clientes existentes.

## Estrutura dos endpoints

### Endpoints do backend

Os endpoints nativos do backend seguem a estrutura:

```text
GET    /api/v1/users              — Lista de usuários
POST   /api/v1/users              — Criar usuário
GET    /api/v1/users/:id          — Detalhe do usuário
PUT    /api/v1/users/:id          — Atualizar usuário
DELETE /api/v1/users/:id          — Excluir usuário
```

### Endpoints de microsserviços

Os endpoints expostos pelos microsserviços são acessíveis através do proxy do backend, sob
o prefixo **`/ws/:serviceCode/`**:

```text
GET    /api/v1/ws/document-service/documents       — Lista de documentos
POST   /api/v1/ws/document-service/documents       — Criar documento
GET    /api/v1/ws/document-service/documents/:id   — Detalhe do documento
```

O `serviceCode` identifica o microsserviço. O backend roteia a requisição para o microsserviço
correspondente com balanceamento round-robin se houver múltiplas instâncias.

## Endpoints públicos

Alguns endpoints não requerem autenticação:

| Endpoint | Descrição |
|----------|-----------|
| `GET /api/v1/health` | Health check do backend |
| `GET /api/v1/openapi.json` | Especificação OpenAPI do backend |
| `GET /api/v1/openapi/aggregated.json` | Especificação OpenAPI agregada (backend + microsserviços online) |

O endpoint OpenAPI agregado é público para permitir que o API Explorer funcione sem autenticação.

## Convenções

### Formato das respostas

Respostas bem-sucedidas retornam o payload diretamente em formato JSON:

```json
{
  "id": "usr_abc123",
  "username": "mrossi",
  "email": "m.rossi@example.com",
  "createdAt": "2025-01-15T10:30:00Z"
}
```

Respostas de erro seguem o padrão **RFC 7807 Problem Details** (ver
[Tratamento de erros](/pt/api/error-handling/)).

### Paginação

Os endpoints que retornam listas suportam paginação através dos query parameters `page` e
`pageSize`:

```text
GET /api/v1/users?page=2&pageSize=20
```

A resposta inclui metadados de paginação:

```json
{
  "data": [...],
  "pagination": {
    "page": 2,
    "pageSize": 20,
    "totalItems": 150,
    "totalPages": 8
  }
}
```

### Filtros e ordenação

Muitos endpoints suportam filtros e ordenação através de query parameters:

```text
GET /api/v1/users?role=admin&sortBy=createdAt&sortOrder=desc
```

## API Explorer

Primebrick inclui um **API Explorer** interativo baseado em Scalar, acessível pela
página [API Explorer](/api-explorer/). O explorer:

- Carrega a especificação OpenAPI agregada no lado do cliente
- Permite configurar a URL do backend (dev, staging, produção)
- Suporta autenticação com bearer token, API key ou OAuth2
- Permite testar os endpoints diretamente do navegador
