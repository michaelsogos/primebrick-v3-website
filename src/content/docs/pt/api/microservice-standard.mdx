## Visão geral

Todo microsserviço do Primebrick segue um **contrato padrão** aplicado pelo `@primebrick/sdk`. Isso garante um comportamento uniforme em todos os serviços — independentemente da lógica de negócio que implementam — e permite ao backend gerenciá-los de forma genérica.

## O contrato padrão

Um microsserviço em conformidade deve:

1. **Autorregistrar-se** no backend ao iniciar
2. **Enviar heartbeats** em intervalos regulares
3. **Expor OpenAPI** em `/api/openapi.json`
4. **Usar NATS** para mensageria entre serviços
5. **Retornar RFC 7807 Problem Details** para todos os erros

## Autorregistro

Ao iniciar, um microsserviço se registra no backend usando o SDK:

```typescript
import { Microservice } from '@primebrick/sdk';

const service = new Microservice({
  serviceCode: 'billing',
  baseUrl: 'http://localhost:3002',
  healthEndpoint: '/health',
  openapiEndpoint: '/api/openapi.json',
  natsUrl: 'nats://localhost:4222',
});

await service.start();
```

O payload de registro inclui:

| Campo | Descrição |
|-------|-----------|
| `serviceCode` | Identificador único do serviço (ex. `billing`) |
| `baseUrl` | URL onde a instância é acessível |
| `healthEndpoint` | Caminho do health check |
| `openapiEndpoint` | Caminho da especificação OpenAPI |

## Heartbeats e detecção de instâncias obsoletas

Cada instância registrada envia um **heartbeat** ao backend em um intervalo configurável (padrão: a cada 15 segundos).

O backend rastreia o último heartbeat de cada instância. Se uma instância não enviar um heartbeat dentro do **limiar de obsolescência** (padrão: 60 segundos), ela é:

1. Marcada como **obsoleta** (stale) no service registry
2. Removida da rotação do **proxy round-robin**
3. Excluída da **especificação OpenAPI agregada**

Isso garante que instâncias travadas ou inacessíveis sejam detectadas automaticamente sem intervenção manual.

## Proxy round-robin

O backend expõe um proxy transparente em **`/ws/:serviceCode`** que encaminha as requisições para as instâncias registradas do microsserviço solicitado.

```text
GET  /ws/billing/invoices        → proxied para o microsserviço billing
POST /ws/inventory/items         → proxied para o microsserviço inventory
```

Quando múltiplas instâncias de um serviço estão online, o proxy distribui as requisições usando balanceamento **round-robin**. Os clientes não precisam saber quantas instâncias existem nem onde estão — eles sempre se conectam ao backend e o proxy gerencia o roteamento.

## Mensageria NATS

Os microsserviços se comunicam entre si e com o backend através do **NATS**. O SDK fornece helpers para padrões comuns:

```typescript
// Publicar um evento
await service.nats.publish('billing.invoice.created', { invoiceId: 123 });

// Assinar eventos
service.nats.subscribe('inventory.item.updated', async (msg) => {
  await updatePricing(msg.data.itemId);
});

// Request/reply (estilo síncrono sobre NATS)
const result = await service.nats.request('inventory.check-stock', {
  itemId: 'SKU-001',
  quantity: 10,
});
```

NATS desacopla os serviços: um serviço pode publicar eventos sem saber quem os consome, e múltiplas instâncias podem assinar o mesmo subject para escalonamento horizontal.

## Exposição OpenAPI

Cada microsserviço deve expor sua especificação OpenAPI em:

```text
GET /api/openapi.json
```

O backend busca a especificação de cada serviço online e a combina na **especificação agregada** em `/api/v1/openapi/aggregated.json`. Os caminhos dos microsserviços são prefixados com `/ws/:serviceCode` para que correspondam ao proxy.

Se um serviço está offline, sua especificação é simplesmente omitida da agregação.

## Tratamento de erros RFC 7807

Todos os erros — do backend e dos microsserviços — usam **RFC 7807 Problem Details**. O SDK fornece um helper para produzir erros conformes:

```typescript
import { ProblemError } from '@primebrick/sdk';

throw new ProblemError({
  type: 'https://primebrick.dev/errors/invoice-not-found',
  title: 'Invoice not found',
  status: 404,
  detail: `Invoice ${invoiceId} does not exist.`,
  instance: `/ws/billing/invoices/${invoiceId}`,
});
```

Ver [Tratamento de erros](./error-handling/) para o formato completo.

## Lista de verificação

Um microsserviço está em conformidade quando:

- [x] Registra-se via `@primebrick/sdk` ao iniciar
- [x] Envia heartbeats e gerencia a detecção de obsolescência
- [x] É acessível através do proxy `/ws/:serviceCode`
- [x] Usa NATS para mensageria entre serviços
- [x] Expõe OpenAPI em `/api/openapi.json`
- [x] Retorna RFC 7807 Problem Details para todos os erros
