# Introducción

## ¿Qué es Primebrick?

**Primebrick** es un framework backoffice **de código abierto** y **opinionado**. Proporciona una
arquitectura completa y predefinida para construir aplicaciones de gestión de datos (backoffice,
paneles de administración, dashboards internos) sin tener que reescribir cada vez los mismos
componentes fundamentales: autenticación, autorización, API REST, capa de acceso a datos y frontend.

Primebrick se distribuye bajo la **licencia MIT**, una de las licencias de código abierto más
permisivas. Puedes usarlo libremente en proyectos personales, comerciales o empresariales,
modificarlo y distribuirlo sin restricciones significativas.

## ¿Por qué "opinionado"?

Un framework "opinionado" toma decisiones arquitectónicas por ti. En lugar de ofrecer un conjunto
genérico de herramientas para componer, Primebrick define:

- **Stack tecnológico fijo**: Express para el backend, SvelteKit para el frontend, NATS para la
  comunicación entre microservicios.
- **Convenciones estrictas**: estructura de la API, formato de errores, estándar de registro de
  microservicios, modelo de autorización.
- **Componentes preintegrados**: Casdoor como Identity Provider, DAL como capa de acceso a datos,
  SDK para la generación de clientes tipados.

Este enfoque reduce drásticamente el tiempo de configuración y garantiza que todos los proyectos
Primebrick sigan las mismas convenciones, simplificando el mantenimiento, la incorporación de nuevos
desarrolladores y las contribuciones de la comunidad.

## Multi-cloud, sin vendor lock-in

Uno de los principios fundamentales de Primebrick es la **ausencia de vendor lock-in**. Todo el
stack está diseñado para funcionar en cualquier sitio:

- **Cloud público**: AWS, Google Cloud, Azure, Cloudflare Workers
- **Cloud privado**: cualquier infraestructura Kubernetes o Docker
- **On-premise**: servidores tradicionales con Docker o bare metal

No existen dependencias de servicios gestionados específicos de un proveedor. La base de datos, el
message broker (NATS), el Identity Provider (Casdoor) y el runtime son todos componentes de código
abiertos autoalojables. Puedes mover una instalación de Primebrick de un cloud a otro sin reescribir
código de la aplicación.

## Qué incluye Primebrick

| Componente | Descripción |
|------------|-------------|
| **Backend** | API REST en Express con autenticación OIDC, RBAC y proxy para microservicios |
| **Frontend** | Aplicación SvelteKit con UI de backoffice preconfigurada |
| **Microservicios** | Estándar para servicios autónomos con autoregistro y heartbeat |
| **DAL** | Data Access Layer — ORM abstracto con soporte multi-base de datos |
| **SDK** | Generación automática de clientes tipados a partir de las especificaciones OpenAPI |
| **Casdoor** | Identity Provider para autenticación OIDC, gestión de usuarios y roles |
