# Inicio rápido

Esta guía te acompaña en la puesta en marcha de un entorno de desarrollo Primebrick completo en pocos minutos.

## Requisitos previos

Antes de empezar, asegúrate de tener instaladas las siguientes herramientas:

| Herramienta | Versión mínima | Notas |
|-------------|----------------|-------|
| **Node.js** | 20+ | Verifica con `node --version` |
| **pnpm** | 9+ | Verifica con `pnpm --version` |
| **Docker** | 24+ | Necesario para los servicios de infraestructura (base de datos, NATS, Casdoor) |
| **Git** | 2.40+ | Para clonar los repositorios |

### Instalación de los requisitos previos

Si aún no tienes `pnpm`, puedes instalarlo con:

```bash
npm install -g pnpm
```

Verifica que Docker esté en ejecución:

```bash
docker info
```

## 1. Clonar los repositorios

Primebrick está compuesto por varios repositorios. Para un entorno de desarrollo completo, clona el
monorepo principal o los repositorios individuales:

```bash
# Clonar el backend
git clone https://github.com/michaelsogos/primebrick-be-v3.git

# Clonar el frontend
git clone https://github.com/michaelsogos/primebrick-fe-v3.git

# Clonar los microservicios (opcional para empezar)
git clone https://github.com/michaelsogos/primebrick-us-v3.git
```

## 2. Instalar las dependencias

Entra en cada repositorio e instala las dependencias con `pnpm`:

```bash
cd primebrick-be-v3
pnpm install

cd ../primebrick-fe-v3
pnpm install
```

## 3. Iniciar los servicios de infraestructura

El backend requiere algunos servicios de infraestructura (base de datos, NATS, Casdoor). Estos están
definidos en un archivo `docker-compose.yml` y pueden iniciarse con Docker:

```bash
cd primebrick-be-v3
docker compose up -d
```

Esto inicia en segundo plano:

- **PostgreSQL** — base de datos principal
- **NATS** — message broker para la comunicación entre microservicios
- **Casdoor** — Identity Provider para autenticación OIDC

Verifica que todos los servicios estén activos:

```bash
docker compose ps
```

## 4. Configurar las variables de entorno

Copia el archivo `.env.example` a `.env` y modifica los valores si es necesario:

```bash
cp .env.example .env
```

Las variables principales incluyen:

```bash
# Base de datos
DATABASE_URL=postgresql://primebrick:primebrick@localhost:5432/primebrick

# NATS
NATS_URL=nats://localhost:4222

# Casdoor
CASDOOR_ENDPOINT=http://localhost:8000
CASDOOR_CLIENT_ID=xxx
CASDOOR_CLIENT_SECRET=xxx
```

## 5. Ejecutar las migraciones de la base de datos

Aplica el esquema de la base de datos ejecutando las migraciones:

```bash
pnpm run db:migrate
```

## 6. Iniciar el backend

```bash
pnpm run dev
```

El backend estará disponible en `http://localhost:3001`.

Verifica que la API responda:

```bash
curl http://localhost:3001/api/v1/health
```

## 7. Iniciar el frontend

En una nueva terminal:

```bash
cd primebrick-fe-v3
pnpm run dev
```

El frontend estará disponible en `http://localhost:5173`.

## Verificación de la instalación

Abre el navegador en `http://localhost:5173`. Deberías ver la página de login de Primebrick.
Usa las credenciales por defecto de Casdoor para acceder al backoffice.

## Resolución de problemas

### Docker no inicia los servicios

Asegúrate de que Docker Desktop esté en ejecución y de que los puertos `5432` (PostgreSQL), `4222`
(NATS) y `8000` (Casdoor) no estén ya ocupados por otros procesos.

### El backend no se conecta a la base de datos

Verifica que PostgreSQL esté activo y que la `DATABASE_URL` en el archivo `.env` sea correcta.

### Error "pnpm not found"

Instala pnpm globalmente con `npm install -g pnpm` o usa `corepack enable pnpm`.
