# Introduzione

## Cos'è Primebrick?

**Primebrick** è un framework backoffice **open source** e **opinionato**. Fornisce un'architettura
completa e predefinita per costruire applicazioni di gestione dati (backoffice, admin panel,
dashboard interne) senza dover riscrivere ogni volta le stesse componenti fondamentali:
autenticazione, autorizzazione, API REST, layer di accesso ai dati e frontend.

Primebrick è distribuito sotto licenza **MIT**, una delle licenze open source più permissive.
Puoi utilizzarlo liberamente in progetti personali, commerciali o enterprise, modificarlo e
distribuirlo senza restrizioni significative.

## Perché "opinionato"?

Un framework "opinionato" prende decisioni architetturali al posto tuo. Invece di offrire
un insieme generico di strumenti da comporre, Primebrick definisce:

- **Stack tecnologico fisso**: Express per il backend, SvelteKit per il frontend, NATS per la
  comunicazione tra microservizi.
- **Convenzioni rigide**: struttura delle API, formato degli errori, standard di registrazione
  dei microservizi, modello di autorizzazione.
- **Componenti pre-integrati**: Casdoor come Identity Provider, DAL come layer di accesso ai
  dati, SDK per la generazione di client tipizzati.

Questo approccio riduce drasticamente il tempo di setup e garantisce che tutti i progetti
Primebrick seguano le stesse convenzioni, semplificando manutenzione, onboarding e contributi
dalla community.

## Multi-cloud, nessun vendor lock-in

Uno dei principi fondamentali di Primebrick è l'**assenza di vendor lock-in**. L'intero stack
è progettato per girare ovunque:

- **Cloud pubblico**: AWS, Google Cloud, Azure, Cloudflare Workers
- **Cloud privato**: qualsiasi infrastruttura Kubernetes o Docker
- **On-premise**: server tradizionali con Docker o bare metal

Non esistono dipendenze da servizi gestiti specifici di un provider. Il database, il message
broker (NATS), l'Identity Provider (Casdoor) e il runtime sono tutti componenti open source
auto-ospitabili. Puoi spostare un'installazione Primebrick da un cloud all'altro senza riscrivere
codice applicativo.

## Cosa include Primebrick

| Componente | Descrizione |
|-----------|-------------|
| **Backend** | API REST in Express con autenticazione OIDC, RBAC e proxy per microservizi |
| **Frontend** | Applicazione SvelteKit con UI di backoffice preconfigurata |
| **Microservizi** | Standard per servizi autonomi con autoregistrazione e heartbeat |
| **DAL** | Data Access Layer — ORM astratto con supporto multi-database |
| **SDK** | Generazione automatica di client tipizzati a partire dalle specifiche OpenAPI |
| **Casdoor** | Identity Provider per autenticazione OIDC, gestione utenti e ruoli |
