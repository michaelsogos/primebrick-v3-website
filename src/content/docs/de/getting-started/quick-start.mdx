# Schnellstart

Diese Anleitung zeigt, wie Sie Primebrick in wenigen Minuten lokal zum Laufen bringen.

## Voraussetzungen

Stellen Sie sicher, dass die folgenden Werkzeuge installiert sind:

| Werkzeug | Mindestversion | Zweck |
|----------|---------------|------|
| **Node.js** | 20+ | JavaScript-Runtime für Backend und Frontend |
| **pnpm** | 9+ | Paketmanager (Monorepo-Unterstützung) |
| **Docker** | 24+ | Container für PostgreSQL, NATS und andere Dienste |

### Node.js installieren

Laden Sie Node.js 20 oder neuer von [nodejs.org](https://nodejs.org/) herunter oder verwenden Sie einen Versionsmanager:

```bash
# Mit nvm (Node Version Manager)
nvm install 20
nvm use 20

# Mit fnm (Fast Node Manager)
fnm install 20
fnm use 20
```

### pnpm installieren

```bash
# Mit Corepack (empfohlen)
corepack enable
corepack prepare pnpm@latest --activate

# Oder mit npm
npm install -g pnpm
```

### Docker installieren

Laden Sie Docker Desktop von [docker.com](https://www.docker.com/products/docker-desktop/) herunter und installieren Sie es.

## Repositories klonen

Primebrick besteht aus mehreren Repositories. Klonen Sie die Hauptkomponenten:

```bash
# Backend (Express API)
git clone https://github.com/michaelsogos/primebrick-be-v3.git

# Frontend (SvelteKit)
git clone https://github.com/michaelsogos/primebrick-fe-v3.git

# Microservices
git clone https://github.com/michaelsogos/primebrick-us-v3.git

# DAL (Data Access Layer)
git clone https://github.com/michaelsogos/primebrick-dal-v3.git

# SDK
git clone https://github.com/michaelsogos/primebrick-v3-sdk.git
```

## Infrastruktur starten

Starten Sie die benötigten Dienste (PostgreSQL, NATS, Casdoor) mit Docker:

```bash
cd primebrick-be-v3

# Docker-Container starten
docker compose up -d
```

Dies startet:

- **PostgreSQL** – Datenbank für das Backend
- **NATS** – Message Broker für Microservice-Kommunikation
- **Casdoor** – Identity Provider für Authentifizierung

## Backend installieren und ausführen

```bash
cd primebrick-be-v3

# Abhängigkeiten installieren
pnpm install

# Datenbank-Migrationen ausführen
pnpm run migrate

# Entwicklungsserver starten
pnpm run dev
```

Das Backend ist nun unter `http://localhost:3001` erreichbar.

## Frontend installieren und ausführen

```bash
cd primebrick-fe-v3

# Abhängigkeiten installieren
pnpm install

# Entwicklungsserver starten
pnpm run dev
```

Das Frontend ist nun unter `http://localhost:5173` erreichbar.

## Microservice starten (optional)

```bash
cd primebrick-us-v3

# Abhängigkeiten installieren
pnpm install

# Microservice starten
pnpm run dev
```

Der Microservice registriert sich automatisch beim Backend und ist über den Proxy erreichbar.

## Verifikation

Öffnen Sie die folgenden URLs im Browser, um zu überprüfen, ob alles funktioniert:

- **Backend Health**: `http://localhost:3001/api/v1/health`
- **Frontend**: `http://localhost:5173`
- **OpenAPI-Spec**: `http://localhost:3001/api/v1/openapi.json`
