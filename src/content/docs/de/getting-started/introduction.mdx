# Einführung

**Primebrick** ist ein opinionated Open-Source-Backoffice-Framework, das Entwicklern eine vollständige Plattform für die Erstellung von Backoffice-Anwendungen bietet. Es wird unter der **MIT-Lizenz** veröffentlicht und ist frei verwendbar, modifizierbar und verteilbar.

## Was ist ein "opinionated" Framework?

Ein opinionated Framework trifft Entscheidungen für Sie. Anstatt jede Komponente einzeln auszuwählen und zu konfigurieren, liefert Primebrick einen klar definierten Satz an Technologien, Mustern und Konventionen mit:

- **Vorgegebene Architektur**: Express API im Backend, SvelteKit im Frontend, Microservices nach Standard.
- **Vorgegebene Konventionen**: Dateistruktur, Namensgebung, API-Design und Fehlerbehandlung folgen festen Regeln.
- **Integrierte Best Practices**: Authentifizierung, Autorisierung, Datenzugriff und Service-Kommunikation sind bereits gelöst.

Dies reduziert den Entscheidungsaufwand und ermöglicht es Teams, sich auf die Geschäftslogik zu konzentrieren.

## Hauptmerkmale

### Multi-Cloud

Primebrick ist cloud-agnostisch. Es kann auf jeder Cloud-Plattform betrieben werden – sei es AWS, Azure, Google Cloud oder eine On-Premise-Infrastruktur. Es gibt keine Abhängigkeit von herstellerspezifischen Cloud-Diensten.

### Kein Vendor Lock-in

Da Primebrick vollständig Open-Source ist und auf Standard-Technologien basiert, sind Sie nicht an einen bestimmten Anbieter gebunden:

- **Datenbank**: PostgreSQL als Standard, aber austauschbar.
- **Message Broker**: NATS als Standard, aber austauschbar.
- **Identity Provider**: Casdoor als Standard, OIDC-kompatibel.
- **Hosting**: Beliebige Cloud oder On-Premise.

### MIT-Lizenz

Die MIT-Lizenz ist eine der freizügigsten Open-Source-Lizenzen:

- Kommerzielle Nutzung erlaubt
- Modifikation erlaubt
- Verbreitung erlaubt
- Private Nutzung erlaubt
- Keine Haftung oder Gewährleistung

### Microservice-Architektur

Primebrick verwendet eine Microservice-Architektur mit Selbstregistrierung. Microservices melden sich beim Backend an, senden regelmäßige Heartbeats und werden über einen Round-Robin-Proxy erreichbar gemacht.

### Datenabstraktion (DAL)

Die Data Access Layer (DAL) bietet eine einheitliche Schnittstelle für den Datenzugriff, die datenbankunabhängig funktioniert und ORM-Funktionen sowie Migrationen bereitstellt.

### SDK

Das Primebrick SDK stellt Typdefinitionen, Client-Bibliotheken und Hilfsfunktionen für die Integration mit der Primebrick-Plattform bereit.
